# AGENTS.md

This file provides context for AI coding assistants (Kiro, Cursor, GitHub Copilot, Claude Code, etc.) working with this repository.

## Your role

You are an **AWS CDK (Cloud Development Kit) and TypeScript** expert. This project is **ABCA (Autonomous Background Coding Agents on AWS)**: a self-hosted platform where users create background coding agents, submit coding tasks, and the agents work autonomously in isolated cloud environments ‚Äî cloning repos, writing code, running tests, and opening pull requests for review. The codebase is CDK infrastructure (TypeScript) plus Python agent code that runs inside the compute environment.

## Project knowledge

To get started and understand the developer flow, follow the [Developer guide](./docs/guides/DEVELOPER_GUIDE.md). For architecture and design, see [docs/design/](./docs/design/ARCHITECTURE.md).

### Tech stack

- **Language / runtime** ‚Äî TypeScript (Node 20.x‚Äì24.x), Python 3.9+ (agent code in `agent/`)
- **Infrastructure** ‚Äî AWS CDK v2 (awscdk), CDK constructs v10.x
- **CDK / AWS** ‚Äî `@aws-cdk/aws-bedrock-alpha`, `@aws-cdk/aws-bedrock-agentcore-alpha`, `cdk-nag`

### Repository structure

- **`.projenrc.ts`** ‚Äî Projen project definition (deps, eslint, scripts). Edit this to add deps or change project config. Defines both the root CDK project and the `@backgroundagent/cli` subproject.
- **`src/`** ‚Äî CDK app entry and stacks (TypeScript). `main.ts` is the app entry; `stacks/` contains CDK stacks (e.g. agent stack).
- **`cli/`** ‚Äî `@backgroundagent/cli` ‚Äî CLI tool for interacting with the deployed REST API (see below).
- **`agent/`** ‚Äî Python code that runs inside the agent compute environment (entrypoint, server, system prompt, Dockerfile, requirements).
- **`docs/`** ‚Äî Authoritative Markdown in `guides/` (developer, user, roadmap, prompt) and `design/`; assets in `diagrams/`, `imgs/`. The Starlight docs site is a Projen subproject here (`astro.config.mjs`, `package.json`); `src/content/docs/` is generated from `.projenrc.ts` on `npx projen`.
- **`CONTRIBUTING.md`** ‚Äî Contribution guidelines at the repository root.
- **`test/`** ‚Äî Jest tests for the CDK app.
- **`cdk.json`** ‚Äî CDK CLI configuration (generated by projen).
- **`package.json`**, **`yarn.lock`** ‚Äî Dependencies and lockfile (package.json is partially managed by projen).

### CLI package (`cli/`)

The `@backgroundagent/cli` package is a projen `TypeScriptProject` subproject defined in `.projenrc.ts`. It provides the `bgagent` executable for submitting and managing tasks through the deployed REST API with Cognito authentication.

**Structure:**
- `src/bin/bgagent.ts` ‚Äî Entry point (`#!/usr/bin/env node`, commander program setup)
- `src/commands/` ‚Äî One file per command: `configure`, `login`, `submit`, `list`, `status`, `cancel`, `events`
- `src/api-client.ts` ‚Äî HTTP client wrapping `fetch` with auth header injection
- `src/auth.ts` ‚Äî Cognito login, token caching (`~/.bgagent/credentials.json`), auto-refresh
- `src/config.ts` ‚Äî Read/write `~/.bgagent/config.json`
- `src/types.ts` ‚Äî API request/response types (mirrored from `src/handlers/shared/types.ts`)
- `src/format.ts` ‚Äî Output formatting (table, detail view, JSON)
- `src/debug.ts` ‚Äî Verbose/debug logging (`--verbose` flag)
- `src/errors.ts` ‚Äî `CliError` and `ApiError` classes
- `test/` ‚Äî Jest tests for all modules

**Key conventions:**
- The `no-console` ESLint rule is disabled for CLI source files (console output is the product).
- Runtime deps (`commander`, `@aws-sdk/client-cognito-identity-provider`) are declared in `.projenrc.ts` under the CLI subproject's `deps`.
- The CLI build is hooked into the root build pipeline via `project.postCompileTask`, so `npx projen build` from root covers both projects.
- The API URL from the `ApiUrl` stack output already includes the stage name (`/v1/`), so the CLI appends only resource paths (`/tasks`, `/tasks/{id}`, etc.).

## Commands you can use

Run `npx projen --help` for the full list. Common commands:

- **`npx projen install`** ‚Äî Install dependencies and sync project files from .projenrc.ts.
- **`npx projen build`** ‚Äî Full release build (compile, test, synth).
- **`npx projen compile`** ‚Äî Compile TypeScript only.
- **`npx projen test`** ‚Äî Run Jest tests.
- **`npx projen synth`** ‚Äî Synthesize CDK app to `cdk.out/`.
- **`npx projen deploy`** ‚Äî Deploy the CDK stack to the current AWS account.
- **`npx projen destroy`** ‚Äî Destroy the deployed CDK stack.
- **`npx projen diff`** ‚Äî Diff deployed stack vs. current code.
- **`npx projen watch`** ‚Äî Watch source and rebuild/redeploy on changes.
- **`npx projen eslint`** ‚Äî Run ESLint.
- **`npx projen upgrade`** ‚Äî Upgrade dependencies.

Use these instead of running `tsc`, `jest`, or `cdk` directly when possible, so the project's scripts and config stay consistent.

To build or test only the CLI subproject:

- **`cd cli && npx projen build`** ‚Äî Full CLI build (compile, test, lint, package).
- **`cd cli && npx projen test`** ‚Äî Run CLI tests only.
- **`cd cli && npx projen compile`** ‚Äî Compile CLI TypeScript only.

## Boundaries

- **Projen-managed files** ‚Äî Do not edit files that contain "Generated by projen" (or that are clearly generated, e.g. from `.projenrc.ts`). Use projen commands or change `.projenrc.ts` and run `npx projen` to update the project.
- **Dependencies** ‚Äî Do not add new dependencies without adding them in `.projenrc.ts` (in `deps` or `devDeps`). Ask first if unsure.
- **Build before commit** ‚Äî Run a full build (`npx projen build`) when done with a task so generated files and tests are up to date before commit.
- **Major changes** ‚Äî Before modifying existing files in a major way (large refactors, new stacks, changing the agent contract), ask first.
